# Track Page Refresh — screenshots

Generated by **`tests/e2e/track-page.spec.js`**. They could not be captured in
the headless QA sandbox (no browser binary / missing host libraries), so run
them locally:

```bash
# start the backend + seed so a real track exists, then:
cd backend && npm run db:seed && cd ..
npm run test:e2e -- track-page
```

Expected files:

| File | Viewport / theme |
|---|---|
| `desktop-track-1440x900.png` | 1440×900 |
| `mobile-track-390x844.png` | 390×844 |
| `light-theme-track.png` | Light Minimal |
| `green-stream-track.png` | Green Stream |
| `orange-wave-track.png` | Orange Wave |

The real-track screenshots require a published track (backend seeded). Without a
backend the spec still passes — it asserts the routing + localized error states
and skips the data-dependent captures.
