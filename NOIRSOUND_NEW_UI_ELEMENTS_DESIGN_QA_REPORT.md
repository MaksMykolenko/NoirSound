## 6. Issues found

| # | Severity | Area | Issue |
|---|---|---|---|
| 1 | High | GenrePicker / mobile | Picker opened as an absolute `z-30` dropdown — on mobile it could be covered by the mini-player (z-30) and bottom nav (z-40), and a 288px dropdown could run off short screens. |
| 2 | High | Discover chip | The selected-genre chip nested an icon-only `aria-label` button; the global mobile rule `button[aria-label]{min-width:44px}` stretched that inner button, breaking the chip on phones. |
| 3 | Med | GenrePicker / a11y | No arrow-key navigation; `role="listbox"` wrapped the search input (wrong semantics); search autofocus fired on mobile, popping the keyboard over the sheet; no explicit close control. |
| 4 | Med | i18n / overflow | Long localized genre labels could overflow the track-card badge, the track-row badge, and the stats breakdown row. |
| 5 | Low | Discover | "More" was visually identical to group tabs (not obviously a disclosure). |
| 6 | Low | Discover / mobile | Quick-tab row was a little dense at 360px. |

## 7. Fixes made (summary)

1. **GenrePicker → responsive sheet.** Mobile (<640px) now renders a **bottom sheet** at `z-[60]` with a `z-[55]` backdrop, `max-h-[80vh]`, internal scrolling list (`max-h-[56vh]`), safe-area padding, and a header with a close button — so it sits above the nav/player and never clips. Desktop keeps the anchored dropdown (`z-40`).
2. **GenrePicker → accessibility.** Arrow Up/Down/Home/End roving focus across options, Escape closes and returns focus to the trigger, desktop-only search autofocus, `role="listbox"` scoped to the options only, `aria-selected` on options, `aria-label`'d close button, larger mobile tap rows.
3. **Discover chip** is now a single accessible button (`aria-label="Clear genre: <label>"`) with a truncating label — no inner 44px stretch, clean on mobile.
4. **Long-label hardening:** truncation + bounded containers on the track-card badge, track-row badge, and stats breakdown row.
5. **"More" disclosure** gets a rotating chevron to read as a disclosure without dominating.
6. **Quick tabs** use slightly tighter padding/size on mobile (still ≥44px tap height) and continue to wrap (no horizontal scroll).

Details and before/after in `NOIRSOUND_GENRE_UI_DESIGN_FIX_REPORT.md`.

## 9. Remaining UI issues / notes

- Real screenshots require running the e2e spec locally (no browser in the QA sandbox).
- On mobile, opening the full taxonomy is two taps (tap **More**, then the field). A future tweak could auto-open the sheet when "More" is tapped.
- iOS Safari can overlay the keyboard on a focused bottom-sheet search; mitigated by placing search at the top of the sheet and not autofocusing on mobile.
- `npm run build` / `npm test` / `npm run test:e2e` were not executed in-sandbox — they are the final sign-off step in your environment.
